Проанализируй сообщение пользователя и извлеки его намерение(я) и именованные сущности.

Верни результат анализа в виде JSON строго следующей структуры:
{
"intents": [
{ "name": "название_намерения", "confidence": 0.95, "attributes": {} }
],
"entities": [
{ "type": "тип_сущности", "value": "значение", "metadata": {} }
],
"uncertain": false,
"reasoningSuggestion": "Краткое описание того, что необходимо выполнить"
}

Правила:

- Разбивай на НЕСКОЛЬКО намерений ТОЛЬКО если пользователь явно формулирует несколько НЕЗАВИСИМЫХ целей в одном
  сообщении (например, «забронируй рейс И поставь напоминание»). Единая задача с подшагами (например, «напиши скрипт и
  запусти его») — это ОДНО намерение.
- При сомнении в том, делить или нет — по умолчанию ОДНО намерение.
- confidence — число от 0 до 1.
- Типы сущностей: date, time, city, person, product, number и т.д.
- Если намерение неясно — установи uncertain: true, но всё равно укажи одно наиболее вероятное намерение.
- reasoningSuggestion должен быть чётким, действенным описанием цели.
