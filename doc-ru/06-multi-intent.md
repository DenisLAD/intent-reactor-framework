# Мультинамерения

## Условие срабатывания

Мультинамерения активируются, когда `IntentPreprocessor` обнаруживает более одного намерения в сообщении пользователя.

Пример: *«Сравни наши цены с конкурентами и обнови прайс-лист»* — может быть разбито на два намерения:
1. `compare_prices` — поиск цен конкурентов
2. `update_price_list` — запись в хранилище данных

---

## Стратегии диспетчеризации

Настраивается через `planning.multi-intent.strategy`:

### sequential (по умолчанию)

Намерения выполняются по очереди в порядке убывания уровня уверенности. Каждое намерение проходит полный цикл ReACT. История сессии общая — результат каждого намерения виден при обработке следующего.

Поддерживает полный поток подтверждения: если намерение требует подтверждения, вся последовательность приостанавливается.

### parallel

Для каждого намерения создаётся клон `SessionState`. Все намерения обрабатываются одновременно в `CompletableFuture.allOf(...)` с ограничением по времени `planning.parallel-timeout`.

```yaml
intent-reactor:
  planning:
    parallel-timeout: PT5M  # таймаут для параллельного выполнения
```

**Ограничение:** подтверждение во время параллельного выполнения не поддерживается. Рискованные инструменты в режиме `parallel` требуют `autonomous: true` или должны быть недоступны в сессии.

### llm-driven

LLM получает список намерений и самостоятельно определяет порядок выполнения с обоснованием. При сбое определения порядка — фallback на сортировку по уровню уверенности с последовательным выполнением.

---

## Структура MultiIntentContext

```java
public class MultiIntentContext {
    List<Intent> intents;
    String strategy;                  // "sequential" / "parallel" / "llm-driven"
    Map<String, String> results;      // имя намерения → результат
    Map<String, String> errors;       // имя намерения → сообщение об ошибке
    int currentIntentIndex;           // для sequential
}
```

Хранится в `session.attributes["multiIntentContext"]`.

---

## Слияние результатов

Результаты всех намерений объединяются в финальный ответ в формате:

```
[compare_prices] Найдено 5 конкурентов. Наши цены на 12% выше среднего.
[update_price_list] Прайс-лист обновлён: изменено 47 позиций.
```

---

## Пример

```java
// Одно сообщение — два намерения
String response = reactor.process("session-1",
    "Проверь свободные слоты в моём календаре и запланируй встречу с Алексом на завтра.");

// Стратегия sequential:
// 1. check_calendar → находит свободные слоты
// 2. schedule_meeting → создаёт событие с учётом результатов шага 1
```

---

## Конфигурация

```yaml
intent-reactor:
  planning:
    multi-intent:
      strategy: sequential   # sequential | parallel | llm-driven
    parallel-timeout: PT5M   # только для parallel
```
