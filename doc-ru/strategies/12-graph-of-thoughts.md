# Стратегия Graph of Thoughts (GoT)

**Модуль:** `intent-reactor-strategies`
**Значение стратегии:** `got`

---

## Краткое описание

Graph of Thoughts (граф мыслей) расширяет древовидное рассуждение, позволяя мыслям **сливаться**: несколько путей рассуждения могут быть объединены в одну совокупную мысль, синтезирующую их идеи. Это делает GoT особенно мощным для задач, требующих синтеза, ранжирования или консолидации с нескольких точек зрения — проблем, где конвергенция столь же важна, как и исследование.

---

## Принцип работы

1. **Инициализация** — цель становится корневым узлом ориентированного графа.
2. **Раскрытие** — из каждого активного узла LLM генерирует `num-thoughts` альтернативных следующих мыслей.
3. **Оценка** — каждый узел-мысль получает оценку.
4. **Агрегация (слияние)** — если несколько высокооцениваемых мыслей близки по смыслу, LLM объединяет их в единый синтезированный узел. Это ключевое отличие от ToT — мысли могут конвергировать, а не только ветвиться.
5. **Итерация** — шаги 2–4 повторяются до `max-iterations`.
6. **Финальный ответ** — узел с наивысшей оценкой (или объединённый узел) даёт финальный ответ.

---

## Конфигурация

```yaml
intent-reactor:
  planning:
    strategy: got
    got:
      num-thoughts: 3       # мыслей на активный узел
      max-iterations: 5     # максимум раундов расширения графа
    max-steps: 50
```

## Промпты

| Промпт | Путь в classpath | Назначение |
|---|---|---|
| Промпт генерации | `prompts/got-generate.md` | Генерирует `num-thoughts` альтернатив на узел |
| Промпт оценки | `prompts/got-evaluate.md` | Оценивает каждую мысль |
| Промпт агрегации | `prompts/got-aggregate.md` | Объединяет связанные мысли в одну |

---

## Когда использовать

- **Суммаризация** — исследование разных аспектов, затем слияние в одно связное резюме.
- **Ранжирование / сравнение** — генерация критериев оценки с нескольких точек зрения, затем консолидация.
- **Генерация отчётов** — несколько исследовательских потоков, которые должны в итоге сойтись.
- Любая задача, где разнообразное исследование с последующим синтезом даёт лучшие результаты, чем любой одиночный путь.

---

## Отличие от ToT

| | `tot` | `got` |
|---|---|---|
| Структура графа | Дерево (только ветвление) | Ориентированный граф (ветвление + слияние) |
| Конвергенция | Не поддерживается | Да — мысли могут сливаться |
| Лучше всего для | Исследование, творчество | Синтез, суммаризация, ранжирование |
| LLM-вызовы | `O(num-thoughts × beam-width × depth)` | `O(num-thoughts × max-iterations)` + вызовы слияния |

## Пример

```yaml
intent-reactor:
  planning:
    strategy: got
    got:
      num-thoughts: 3
      max-iterations: 4
```

Цель: *«Сравни PostgreSQL и MongoDB для приложения социальной сети.»*

Итерация 1 — 3 сгенерированные мысли:
- *«Преимущества PostgreSQL: ACID, связи, сложные запросы»*
- *«Преимущества MongoDB: гибкость схемы, горизонтальное масштабирование»*
- *«Требования соцсети: высокий объём записей, графы пользователей, гибкая схема постов»*

Итерация 2 — агрегация объединяет все три: *«Для соцсетей гибкость MongoDB подходит для постов/медиа; PostgreSQL — для связей пользователей и аналитики.»*

Финальный ответ: консолидированное сравнение с рекомендацией.
