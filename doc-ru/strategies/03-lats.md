# Стратегия LATS

**Модуль:** `intent-reactor-core` (всегда доступен)
**Значение стратегии:** `lats`

---

## Краткое описание

LATS (Language Agent Tree Search) объединяет поиск по дереву Монте-Карло с рассуждением LLM. Вместо следования единственной траектории стратегия одновременно исследует `num-candidates` альтернативных следующих шагов, симулирует каждый (используя `SimulatableTool` где возможно), оценивает их и выбирает путь с наивысшей оценкой. Результат — поиск «по лучшему первому» в пространстве возможных последовательностей действий.

---

## Принцип работы

1. **Раскрытие** — на каждом шаге LLM генерирует `num-candidates` альтернативных предложений `PlanStep`. Каждое — отдельное действие, которое может предпринять агент.
2. **Симуляция** — для каждого кандидата-шага с инструментом:
   - Если инструмент реализует `SimulatableTool`, вызывается метод `simulate()` вместо реального инструмента. Симуляция быстрая и без побочных эффектов.
   - Если симуляция недоступна — отдельный вызов LLM предсказывает вероятный результат инструмента.
3. **Оценка** — LLM (или отдельный вызов оценщика) выставляет каждой симулированной траектории оценку от 0.0 до 1.0: насколько она вероятно достигнет цели.
4. **Выбор** — кандидат с наивысшей оценкой выбирается, реальный инструмент выполняется (с потоком подтверждения при необходимости).
5. **Обратное распространение** — выбранная оценка записывается в узел дерева для будущих оценок (используется при `tree-depth > 1`).
6. Шаги 1–5 повторяются до `DONE`, `FAIL` или исчерпания `max-steps`.

## Конфигурация

```yaml
intent-reactor:
  planning:
    strategy: lats
    lats:
      num-candidates: 3           # параллельных веток на шаг
      min-value-threshold: 0.6    # минимальная оценка для принятия кандидата (0.0–1.0)
      tree-depth: 5               # глубина дерева поиска
    max-steps: 50
```

При `num-candidates: 1` LATS деградирует до стандартного ReACT (полезно для тестирования).

---

## SimulatableTool

Реализуйте `SimulatableTool` для инструментов, которые могут безопасно работать в режиме сухого прогона:

```java
@Component
public class DatabaseTool implements Tool, SimulatableTool {

    @Override
    public ToolResult execute(ToolInput input) {
        // реальная запись в БД
    }

    @Override
    public ToolResult simulate(ToolInput input) {
        // вернуть реалистичный результат без изменения состояния
        return ToolResult.ok("Симуляция: будет затронуто 3 строки.");
    }

    // ...
}
```

## Промпты

| Промпт | Путь в classpath | Назначение |
|---|---|---|
| Системный промпт | `prompts/react-system.md` | Разделяется с планировщиком ReACT |
| Промпт генерации кандидатов | `prompts/lats-expand.md` | Генерирует `num-candidates` альтернатив |
| Промпт оценки | `prompts/lats-evaluate.md` | Оценивает каждую симулированную траекторию |

---

## Когда использовать

- Задачи, где к цели ведут многие пути и первый вариант не является очевидно лучшим.
- Исследовательские задачи: генерация кода, планирование запросов, разработка тестов.
- Сценарии, где неверные вызовы инструментов дорого обходятся (LATS избегает плохих действий через симуляцию).

---

## Сравнение плюсов и минусов

| Преимущества | Недостатки |
|---|---|
| Находит лучшие пути, чем жадный ReACT | `num-candidates × LLM-вызовов` на шаг |
| Избегает необратимых плохих действий через симуляцию | Более высокая латентность |
| Настраиваемая ширина и глубина исследования | Инструменты без `SimulatableTool` требуют доп. LLM-вызова на кандидата |

---

## Пример

```yaml
intent-reactor:
  planning:
    strategy: lats
    lats:
      num-candidates: 2
      min-value-threshold: 0.7
      tree-depth: 3
    max-steps: 30
```

С этими настройками LATS исследует 2 варианта на шаг, требует минимум 70% уверенности для принятия ветки и ищет на глубину не более 3 уровней.
