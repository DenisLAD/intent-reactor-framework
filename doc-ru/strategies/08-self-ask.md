# Стратегия Self-Ask

**Модуль:** `intent-reactor-strategies`
**Значение стратегии:** `self-ask`

---

## Краткое описание

Self-Ask (самовопрошание) разбивает сложный вопрос на последовательность более простых подвопросов, отвечает на каждый (потенциально с использованием инструментов) и объединяет промежуточные ответы в финальный. Стратегия хорошо подходит для многоходового фактологического рассуждения, где каждый шаг зависит от предыдущего.

---

## Принцип работы

1. **Декомпозиция** — LLM получает исходный вопрос и его спрашивают: *«Нужно ли вам сначала ответить на какие-то уточняющие вопросы?»* При положительном ответе LLM перечисляет подвопросы.
2. **Цикл подвопросов** — для каждого подвопроса планировщик выполняет мини-проход ReACT:
   - LLM может ответить из собственных знаний или вызвать инструмент.
   - Ответ на подвопрос сохраняется.
3. **Финальный ответ** — все подвопросы и их ответы собираются в контекст. LLM формирует финальный ответ на исходный вопрос.
4. Выдаётся шаг `DONE`.

Self-Ask — декоратор над ReACT и наследует все возможности работы с инструментами.

---

## Конфигурация

```yaml
intent-reactor:
  planning:
    strategy: self-ask
    max-steps: 50   # общий лимит для всех проходов подвопросов
```

## Промпты

| Промпт | Путь в classpath | Назначение |
|---|---|---|
| Промпт декомпозиции | `prompts/self-ask-decompose.md` | Генерирует подвопросы |
| Промпт ответа | `prompts/react-system.md` | Мини-ReACT для каждого подвопроса |
| Промпт синтеза | `prompts/self-ask-synthesize.md` | Финальный ответ из ответов на подвопросы |

---

## Когда использовать

- Многоходовые фактологические вопросы: *«Кто является генеральным директором компании, производящей продукт, используемый в X?»*
- Исследовательские задачи, требующие последовательного сбора информации.
- Любой сценарий, где ответ на исходный вопрос невозможен без предварительного разрешения зависимостей.

---

## Сравнение плюсов и минусов

| Преимущества | Недостатки |
|---|---|
| Точен на многоходовых рассуждениях | Больше LLM-вызовов, чем у плоского ReACT |
| Подвопросы могут вызывать инструменты | Качество декомпозиции зависит от LLM |
| Прозрачная цепочка рассуждений | Может генерировать лишние подвопросы |

---

## Пример

```yaml
intent-reactor:
  planning:
    strategy: self-ask
```

Вопрос: *«В каком году был создан язык программирования, придуманный изобретателем Всемирной паутины?»*

Подвопрос 1: *«Кто изобрёл Всемирную паутину?»* → *«Тим Бернерс-Ли»*

Подвопрос 2: *«Какой язык программирования создал Тим Бернерс-Ли?»* → *«Он не создавал широко известного языка программирования; предпосылка может быть ошибочной.»*

Финальный ответ: *«Тим Бернерс-Ли не известен созданием языка программирования; он известен как создатель HTTP, HTML и Всемирной паутины.»*
