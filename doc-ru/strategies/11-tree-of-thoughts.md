# Стратегия Tree of Thoughts (ToT)

**Модуль:** `intent-reactor-strategies`
**Значение стратегии:** `tot`

---

## Краткое описание

Tree of Thoughts (дерево мыслей) исследует несколько путей рассуждения одновременно с помощью древовидной структуры. На каждом шаге LLM генерирует `num-thoughts` разных «мыслей»-продолжений. Каждая мысль независимо оценивается. Сохраняются только лучшие `beam-width` мыслей для дальнейшего развития. Такой подход beam-search находит высококачественные цепочки рассуждений, которые жадная стратегия пропустила бы.

---

## Принцип работы

1. **Корень** — исходная цель становится корнем дерева мыслей.
2. **Раскрытие** — из текущего beam (набора активных путей) LLM генерирует `num-thoughts` альтернативных следующих мыслей для каждого пути.
3. **Оценка** — каждая мысль оценивается LLM по шкале 0–10: *«Насколько перспективно это направление рассуждений для решения цели?»*
4. **Выбор (обрезка beam)** — сохраняются лучшие `beam-width` мыслей по всем путям.
5. **Рекурсия** — шаги 2–4 повторяются до `max-depth` уровней.
6. **Извлечение ответа** — при достижении глубины или естественного завершения (`DONE`) лучший оцененный лист даёт финальный ответ.

Вызовы инструментов могут появляться как мысли — *«Нужно прочитать файл config.yaml»* — и выполняются до продолжения этого пути.

---

## Конфигурация

```yaml
intent-reactor:
  planning:
    strategy: tot
    tot:
      num-thoughts: 3    # мыслей на путь на шаг
      max-depth: 5       # максимальная глубина дерева
      beam-width: 2      # путей, сохраняемых после каждой обрезки
    max-steps: 50
```

Приблизительное число LLM-вызовов на шаг: `num-thoughts × beam-width + beam-width` (генерация + оценка).

## Промпты

| Промпт | Путь в classpath | Назначение |
|---|---|---|
| Промпт генерации мыслей | `prompts/tot-generate.md` | Генерирует `num-thoughts` альтернатив |
| Промпт оценки мыслей | `prompts/tot-evaluate.md` | Оценивает каждую мысль по шкале 0–10 |
| Системный промпт | `prompts/react-system.md` | Основа с поддержкой инструментов |

---

## Когда использовать

- Творческие задачи: написание текстов, мозговой штурм, проектирование.
- Открытые рассуждения, где несколько подходов одинаково правдоподобны.
- Задачи, требующие возврата (ToT естественно избегает тупиков).
- Анализ сценариев: *«Каковы последствия X?»*

---

## Сравнение плюсов и минусов

| Преимущества | Недостатки |
|---|---|
| Исследует разнообразные пути рассуждений | Много LLM-вызовов: O(num-thoughts × beam-width × depth) |
| Рано избегает тупиковых путей | Медленно для простых задач |
| Поиск «по лучшему первому» | Высокая стоимость на запрос |

---

## Отличие от LATS

| | `lats` | `tot` |
|---|---|---|
| Фокус | Выбор действия (вызовы инструментов) | Выбор шага рассуждения (мысли) |
| Симуляция | SimulatableTool / предсказание LLM | Только оценка LLM |
| Основное применение | Агентские задачи с активным использованием инструментов | Рассуждение / творческие задачи |

## Пример

```yaml
intent-reactor:
  planning:
    strategy: tot
    tot:
      num-thoughts: 3
      max-depth: 4
      beam-width: 2
```

Цель: *«Спроектируй стратегию кеширования для высоконагруженного API.»*

Мысли глубины 1:
- М1: *«Использовать Redis с TTL-экспирацией»* (оценка: 8)
- М2: *«Внедрить слой CDN»* (оценка: 7)
- М3: *«Использовать LRU-кеш на уровне приложения»* (оценка: 6)

Beam сохраняет М1, М2. Каждая развивается дальше, исследуя компромиссы и детали реализации.
