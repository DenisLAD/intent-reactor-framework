# Стратегия Least-to-Most

**Модуль:** `intent-reactor-strategies`
**Значение стратегии:** `least-to-most`

---

## Краткое описание

Least-to-Most (от простого к сложному) разбивает проблему на упорядоченную последовательность подзадач, начиная с самой простой и постепенно приближаясь к исходному вопросу. Каждая подзадача решается последовательно, при этом предыдущие ответы становятся контекстом для следующей. Такой «строительный» подход особенно эффективен для образовательного контента и прогрессивного решения задач.

---

## Принцип работы

1. **Декомпозиция** — LLM получает исходную задачу и его просят перечислить подзадачи в порядке возрастания сложности. Подзадачи обычно являются предварительными условиями: *чтобы понять Y, нужно знать X*.
2. **Последовательное решение** — каждая подзадача решается по порядку. Ответ на подзадачу N становится частью контекста при решении подзадачи N+1. Инструменты доступны на каждом шаге.
3. **Финальное решение** — после решения всех подзадач LLM решает исходную задачу, используя накопленный контекст.
4. Выдаётся шаг `DONE` с финальным ответом.

---

## Конфигурация

```yaml
intent-reactor:
  planning:
    strategy: least-to-most
    max-steps: 50
```

---

## Промпты

| Промпт | Путь в classpath | Назначение |
|---|---|---|
| Промпт декомпозиции | `prompts/ltm-decompose.md` | Перечисляет подзадачи в порядке сложности |
| Промпт подзадачи | `prompts/react-system.md` | Решает каждую подзадачу (с поддержкой ReACT) |
| Финальный промпт | `prompts/ltm-synthesize.md` | Объединяет все ответы в итоговое решение |

---

## Когда использовать

- Создание образовательного контента: репетиторство, разработка учебных программ.
- Составные задачи: *«Создай REST API для X»* → начать с модели данных, затем эндпоинты, затем авторизация.
- Задачи по программированию, где концепции строятся на предыдущих.
- Любая задача, где прямой ответ на исходный вопрос перегружает LLM.

---

## Отличие от Self-Ask

| | `self-ask` | `least-to-most` |
|---|---|---|
| Порядок подзадач | По зависимости (любой порядок) | По сложности (простые → сложные) |
| Основное применение | Многоходовое фактологическое рассуждение | Скаффолдинг навыков, образовательные задачи |
| Перенос контекста | Да | Да — каждый ответ строит следующий контекст |

---

## Пример

```yaml
intent-reactor:
  planning:
    strategy: least-to-most
```

Задача: *«Реализуй бинарное дерево поиска с операциями вставки и поиска на Java.»*

Подзадача 1 (самая простая): *«Что такое узел бинарного дерева поиска?»*

Подзадача 2: *«Как вставить узел в BST?»*

Подзадача 3: *«Как найти значение в BST?»*

Подзадача 4 (самая сложная): *«Напиши полный Java-класс с обеими операциями»* (с ответами на предыдущие подзадачи как контекстом)
