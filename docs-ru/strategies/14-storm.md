# Стратегия STORM

**Модуль:** `intent-reactor-strategies`
**Значение стратегии:** `storm`

---

## Краткое описание

STORM (Synthesis of Topic Outlines through Retrieval and Multi-perspective questioning) генерирует исчерпывающие статьи или отчёты в стиле Википедии путём симуляции панели экспертных перспектив. Каждый симулированный эксперт задаёт вопросы со своей предметной точки зрения, ответы исследуются (через инструменты или знания LLM), и все инсайты синтезируются в структурированный длинный документ.

---

## Принцип работы

**Фаза 1 — Генерация перспектив**
LLM генерирует `max-perspectives` различных экспертных персонажей, релевантных теме. Для темы *«Влияние изменения климата на сельское хозяйство»* персонажи могут быть: агроном, экономист, политический аналитик, климатолог, фермер.

**Фаза 2 — Экспертные вопросы**
Каждый персонаж независимо генерирует `questions-per-perspective` вопросов со своей точки зрения. Вопросы, как правило, не перекрываются, поскольку каждый эксперт заботится о разных аспектах.

**Фаза 3 — Исследование и ответы**
На каждый вопрос отвечает мини-цикл ReACT. Инструменты (веб-запрос, knowledge_search, чтение файлов) полностью доступны. Ответы записываются по каждому вопросу.

**Фаза 4 — Синтез структуры**
LLM собирает результаты исследования в структурированный план: разделы, заголовки, ключевые тезисы.

**Фаза 5 — Написание статьи**
Каждый раздел пишется последовательно на основе соответствующих пар «вопрос-ответ». Финальный проход создаёт связный, хорошо структурированный документ.

---

## Конфигурация

```yaml
intent-reactor:
  planning:
    strategy: storm
    storm:
      max-perspectives: 5           # количество генерируемых экспертных персонажей
      questions-per-perspective: 3  # вопросов от каждого персонажа
    max-steps: 100   # увеличить для тем с большим количеством вопросов
```

Общее количество LLM-вызовов для исследования: примерно `max-perspectives × questions-per-perspective`. Для длинных отчётов может потребоваться `max-steps` выше 100.

## Промпты

| Промпт | Путь в classpath | Назначение |
|---|---|---|
| Генерация перспектив | `prompts/storm-perspectives.md` | Создаёт экспертных персонажей |
| Генерация вопросов | `prompts/storm-questions.md` | Каждый персонаж задаёт вопросы |
| Исследовательский промпт | `prompts/react-system.md` | Цикл ReACT на каждый вопрос |
| Синтез структуры | `prompts/storm-outline.md` | Строит структуру разделов |
| Написание статьи | `prompts/storm-write.md` | Пишет каждый раздел |

---

## Когда использовать

- Генерация длинных отчётов: исследовательские работы, технические white paper, обзоры.
- Темы, требующие охвата из нескольких предметных областей одновременно.
- Статьи для базы знаний: документация продукта, внутренние вики.
- Любая задача, где важны и широта, и глубина.

---

## Сравнение плюсов и минусов

| Преимущества | Недостатки |
|---|---|
| Исчерпывающее многоперспективное покрытие | Много LLM-вызовов и высокая латентность |
| Структурированный, связный длинный вывод | Дорого для простых тем |
| Естественно интегрирует поиск через инструменты | Требует настройки `max-steps` для глубоких тем |
| Разнообразные точки зрения снижают слепые пятна | Объём вывода может быть очень большим |

## Пример

```yaml
intent-reactor:
  planning:
    strategy: storm
    storm:
      max-perspectives: 4
      questions-per-perspective: 3
    max-steps: 80
```

Тема: *«Влияние больших языковых моделей на разработку программного обеспечения.»*

Персонажи: Разработчик ПО, Технический менеджер, Исследователь, Специалист по этике.

Вопросы (примеры):
- Разработчик: *«Какие задачи разработки больше всего выигрывают от помощи LLM?»*
- Менеджер: *«Каковы метрики производительности после внедрения LLM-инструментов?»*
- Исследователь: *«Каковы текущие ограничения возможностей?»*
- Специалист по этике: *«Каковы проблемы с интеллектуальной собственностью и атрибуцией?»*

Вывод: структурированная статья с разделами «Влияние на производительность», «Текущие возможности», «Ограничения», «Этические аспекты» и «Перспективы».
