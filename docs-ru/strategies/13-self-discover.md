# Стратегия Self-Discover

**Модуль:** `intent-reactor-strategies`
**Значение стратегии:** `self-discover`

---

## Краткое описание

Self-Discover — метакогнитивная стратегия: перед выполнением задачи LLM выбирает и компонует модули рассуждения, наиболее подходящие для конкретной проблемы. Он строит индивидуальную структуру рассуждения, адаптированную к данной задаче, а затем следует этой структуре для получения ответа. Это позволяет одному фреймворку адаптировать подход к решению задач для новых нестандартных ситуаций.

---

## Принцип работы

Self-Discover работает в трёх явных фазах до любого вызова инструментов:

**Фаза 1 — Выбор**
LLM получает цель и каталог атомарных модулей рассуждения (например, *«Разбить на подзадачи»*, *«Рассмотреть крайние случаи»*, *«Подумать об ограничениях»*, *«Использовать аналогическое рассуждение»*). LLM выбирает подмножество, наиболее релевантное текущей задаче.

**Фаза 2 — Адаптация**
Выбранные модули уточняются в задачно-специфичные инструкции. Общие описания модулей становятся конкретными директивами для данной проблемы.

**Фаза 3 — Реализация (структурированное рассуждение)**
Адаптированный план рассуждения выполняется шаг за шагом. Каждый шаг следует адаптированному модулю. После завершения структуры рассуждения LLM синтезирует финальный ответ.

Полный план затем передаётся циклу ReACT, который может вызывать инструменты в фазе реализации.

---

## Конфигурация

```yaml
intent-reactor:
  planning:
    strategy: self-discover
    max-steps: 50
```

Специфических свойств нет. Каталог модулей рассуждения определён в шаблоне промпта.

---

## Промпты

| Промпт | Путь в classpath | Назначение |
|---|---|---|
| Промпт выбора | `prompts/self-discover-select.md` | LLM выбирает релевантные модули рассуждения |
| Промпт адаптации | `prompts/self-discover-adapt.md` | Специализирует модули для текущей задачи |
| Промпт реализации | `prompts/self-discover-implement.md` | Выполняет составной план рассуждения |

---

## Когда использовать

- Новые задачи, где ни одна установленная стратегия не является очевидно лучшей.
- Разнообразные рабочие нагрузки: один развёрнутый агент обрабатывает много разных типов задач.
- Сложные проблемы с множеством взаимодействующих ограничений.
- Исследовательские задачи.

---

## Сравнение плюсов и минусов

| Преимущества | Недостатки |
|---|---|
| Адаптирует собственную стратегию к задаче | 3 LLM-вызова до начала выполнения |
| Нет жёстко заданного шаблона решения | Качество зависит от полноты каталога модулей |
| Силён на новых / нестандартных задачах | Избыточен для рутинных задач |

---

## Пример

```yaml
intent-reactor:
  planning:
    strategy: self-discover
```

Цель: *«Разработай справедливый алгоритм распределения ограниченных вакцин от COVID между городами с разным населением и уровнями риска.»*

Фаза 1 — выбранные модули: *«Разбить на подзадачи»*, *«Рассмотреть интересы нескольких сторон»*, *«Определить ограничения»*, *«Рассуждать о критериях справедливости»*.

Фаза 2 — адаптация: *«Подзадачи: (1) определить справедливость, (2) смоделировать риски населения, (3) предложить формулу распределения, (4) проверить крайние случаи.»*

Фаза 3: каждая подзадача решается последовательно, потенциально используя инструменты для получения демографических данных, затем синтезируется в итоговое предложение алгоритма.
