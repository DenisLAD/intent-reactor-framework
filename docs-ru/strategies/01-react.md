# Стратегия ReACT

**Модуль:** `intent-reactor-core` (всегда доступен)
**Значение стратегии:** `react`

---

## Краткое описание

ReACT (Reason + Act) — стратегия IntentReactor по умолчанию. Она управляет итерационным циклом «Мысль → Действие → Наблюдение»: LLM рассуждает о следующем шаге, вызывает инструмент, наблюдает результат и продолжает до завершения задачи или исчерпания лимита шагов.

---

## Принцип работы

1. Системный промпт строится из активных бинов `PromptContextProvider` и добавляется в историю сообщений.
2. LLM вызывается с полной историей диалога (с учётом ограничений контекстного окна).
3. LLM отвечает `PlanStep` в виде JSON-вызова инструмента. Типы шагов:
   - `REASON` — промежуточное рассуждение (инструмент не вызывается, цикл продолжается).
   - `ACT` — вызов именованного инструмента с параметрами.
   - `DONE` — финальный ответ готов; поле `output` становится ответом пользователю.
   - `FAIL` — задача не может быть выполнена; `output` содержит причину.
4. Для шага `ACT`: если инструмент `isRisky()` и `autonomous=false`, выполнение приостанавливается (`AWAITING_CONFIRMATION`). Иначе инструмент выполняется, `ToolResult` добавляется в историю как SYSTEM-сообщение, цикл продолжается.
5. Цикл повторяется с шага 2 до `DONE`, `FAIL` или исчерпания `max-steps`.

---

## Конфигурация

```yaml
intent-reactor:
  planning:
    strategy: react
    max-steps: 50           # максимум итераций
    autonomous: false       # true = пропускать подтверждения
```

Специфических свойств стратегии нет.

---

## Промпты

| Промпт | Путь в classpath | Назначение |
|---|---|---|
| Системный промпт | `prompts/react-system.md` | Инструктирует LLM по формату ReACT-цикла и доступным инструментам |

Фреймворк подставляет список инструментов и дополнительный контекст (от бинов `PromptContextProvider`) в шаблон во время выполнения.

---

## Когда использовать

- **Выбор по умолчанию** для любого агента с инструментами.
- Чётко определённые задачи, где вывод инструментов даёт достаточный сигнал для продвижения.
- Короткие и средние сессии (< 30 шагов).
- Строительный блок для `reflexion` и `lats` — обе стратегии расширяют ReACT.

---

## Пример

```yaml
intent-reactor:
  planning:
    strategy: react
    max-steps: 30
    autonomous: true   # для CI/batch-окружений
```

```java
String result = intentReactorService.process("session-1",
    "Прочитай файл config.yaml и кратко опиши его содержимое.");
// → Выполняет: read_file("config.yaml") → REASON → DONE
```
