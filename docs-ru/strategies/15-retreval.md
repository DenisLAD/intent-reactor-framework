# Стратегия ReTreVal

**Модуль:** `intent-reactor-strategies`
**Значение стратегии:** `retreval`

---

## Краткое описание

ReTreVal (Reasoning Tree with Validation) — обучения не требующая стратегия, которая строит дерево шагов рассуждения, применяет двойную валидацию к каждому кандидату-узлу и выполняет бэктрекинг при неудаче с инжекцией типизированного контекста ошибки. Успешные паттерны рассуждения накапливаются в памяти сессии и переиспользуются в последующих шагах для повышения согласованности.

---

## Принцип работы

**Фаза 1 — EXPAND (расширение)**
LLM генерирует `candidates-per-step` кандидатов следующего шага с учётом текущей цели, доступных инструментов и накопленных паттернов памяти. Каждый кандидат содержит предлагаемое действие (вызов инструмента или шаг рассуждения) и краткое обоснование.

**Фаза 2 — VALIDATE (валидация)**
Каждый кандидат оценивается дважды:
1. **Самооценка** — кандидат сам оценивает себя по шкале 0.0–1.0. Если оценка превышает порог короткого замыкания (0.8), оценка критиком пропускается.
2. **Оценка критиком** — отдельный промпт-критик оценивает выполнимость, релевантность и риски.

Выбирается кандидат с наибольшей суммарной оценкой. Кандидаты с оценкой ниже `validation-threshold` отклоняются.

**Фаза 3 — EXECUTE (исполнение)**
Выбранный узел исполняется (вызов инструмента или шаг REASON). Наблюдение поглощается обратно в запись узла дерева.

**Фаза 4 — BACKTRACK (бэктрекинг)** *(при неудаче)*
Если исполнение завершилось ошибкой или все кандидаты отклонены, планировщик возвращается к родительскому узлу. Тип неудачи и сообщение об ошибке инжектируются в следующий промпт EXPAND, чтобы LLM не повторил ту же ошибку.

**Фаза 5 — SYNTHESIZE (синтез)**
После достижения цели или превышения лимита глубины дерева все исполненные узлы суммируются в финальный ответ.

## Атрибуты сессии

| Атрибут | Описание |
|---|---|
| `retreval_tree` | `RetrevalTree` — полное дерево рассуждений (JSON) |
| `retreval_phase` | Текущая фаза: `EXPAND` / `SYNTHESIZE` |
| `retreval_frontier` | `List<String>` — ID узлов, готовых к следующему расширению (beam) |
| `retreval_cur_node` | ID узла, вызов инструмента которого выполняется (сбрасывается после получения результата) |
| `retreval_patterns` | JSON-список `List<RetrevalPattern>` — накопленные паттерны успехов и неудач |
| `retreval_backtrack` | Строка `[BACKTRACK: type] описание`, инжектируемая в следующий промпт EXPAND |

## Конфигурация

```yaml
intent-reactor:
  planning:
    strategy: retreval
    strategies:
      retreval:
        max-tree-depth: 4         # лимит глубины дерева до принудительного SYNTHESIZE
        candidates-per-step: 3    # кандидатов, генерируемых на фазе EXPAND
        validation-threshold: 0.6 # минимальная оценка (среднее self+critic) для VALIDATED
        beam-width: 2             # максимальный размер frontier (параллельные ветки)
        final-threshold: 0.75     # порог оценки для немедленной пометки узла DONE
        use-external-critic: true # оценивать каждого кандидата вторым промптом-критиком
        memory-enabled: true      # накапливать паттерны успехов/неудач между шагами
        max-memories: 10          # максимум паттернов в retreval_patterns
    max-steps: 40
```

При `use-external-critic: false` используется только самооценка. Увеличьте `max-steps` для глубоких деревьев: каждый узел требует минимум 2 вызова LLM (expand + оценка).

---

## Когда использовать

- Сложные многошаговые задачи, где обычный ReACT застревает в локальных оптимумах.
- Задачи, где выгодно рассмотреть несколько кандидатов следующего шага перед выбором.
- Сценарии, где прошлые неудачи должны влиять на генерацию будущих шагов.

---

## Компромиссы

| Плюсы | Минусы |
|---|---|
| Систематическое исследование с валидацией | Больше вызовов LLM, чем в чистом ReACT |
| Типизированный бэктрекинг исключает повторение ошибок | Память сессии растёт при длинных диалогах |
| Накопленные паттерны повышают согласованность | `max-steps` требует тонкой настройки для глубоких деревьев |
| Не требует обучения — работает с любым LLM | Сложнее отлаживать, чем линейные стратегии |
